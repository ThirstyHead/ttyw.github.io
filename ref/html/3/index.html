HTML 3.x Specification

Abstract

The HyperText Markup Language (HTML) is a simple markup language used to create hypertext documents that are portable from one platform to another. HTML documents are SGML documents with generic semantics that are appropriate for representing information from a wide range of applications. This specification defines HTML version 3.2. HTML 3.2 aims to capture recommended practice as of early '96 and as such to be used as a replacement for HTML 2.0 (RFC 1866).

[source: HTML 3.2 Reference Specification, published January 1997, https://www.w3.org/TR/2018/SPSD-html32-20180315/]

Elements

Document

Document elements describe the high-level parts of the page. The <html> element is the root element of the page. The <head> element is for hidden, browser-focused metadata. All viewable content is defined in the <body> section.

  • head

  • script

Content

Content elements describe the basic hierarchy of the page. Each heading (<h1>..<h6>) creates its own implicit section. Think of headings as entries in a table of contents.

<h1> is the primary heading/section. There should only be one per page. <h2> sections are subtopics. If you need to further subdivide the sections, use <h3> - <h6>.

  • h2

  • h5
  • h6
  • p
  • div
  • center
  • hr

Action

Action elements are the primary way that users interact with the page. <a> is for creating hyperlinks to other web pages.

List

List elements allow you to present the user with a collection of <li> list items. <ul> is an unordered list, or a list of bulleted items. <ol> is an ordered list, or a list of numbered items.

  • ul
  • li
  • ol
  • dl
  • dt
  • dd
  • dir
  • menu

Form

Form elements allow the user to submit information.

  • form

Table

Table elements are ideal for displaying spreadsheet-like information. If your data can be logically grouped by columns, rows, or both, use a <table>.

  • table

Semantics

Semantic elements should be used to describe or clarify the meaning of the text. The way the text appears is far less important than providing the meaning or significance of the text.

  • address
  • pre
  • xmp
  • listing
  • plaintext
  • blockquote

Graphics

Graphic elements show non-text information. Use <img> to display raster or photographic images.
