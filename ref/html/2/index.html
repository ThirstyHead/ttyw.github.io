HTML 2.x Specification

5. Document Structure

An HTML document is a tree of elements, including a head and body, headings, paragraphs, lists, etc.

[source: RFC 1866, published November 1995, https://www.ietf.org/rfc/rfc1866.txt]

Elements

HTML Elements are the basic building blocks of any web page.

Document

Document elements describe the high-level parts of the page. The <html> element is the root element of the page. Non-visible elements like <style> and <script> are defined in the <head> section. All viewable content is defined in the <body> section.

Content

Content elements describe the basic hierarchy of the page. Each heading (<h1>..<h6>) creates its own implicit section. Think of headings as entries in a table of contents.

<h1> is the primary heading/section. There should only be one per page. <h2> sections are subtopics. If you need to further subdivide the sections, use <h3> - <h6>.

Action

Action elements are the primary way that users interact with the page. <a> is for creating hyperlinks to other web pages. <button> provides a larger, more obvious enticement for the user to interact with the current page.

List

List elements allow you to present the user with a collection of <li> list items. <ul> is an unordered list, or a list of bulleted items. <ol> is an ordered list, or a list of numbered items.
