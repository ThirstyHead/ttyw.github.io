Exercise: Introduction

In this exercise, you'll install various bits of software, download several base projects (one for the website and another for the tests), and generally get your development environment set up and configured.

Software

Here is the software you should have installed to complete the exercises.

Software URL Description
Node/NPM https://nodejs.org You'll be asked to npm install software, as well as run tasks like npm test. Version 10.x LTS or later.
Gauge https://gauge.org Gauge is a free and open source test runner for the User Journey tests we'll be writing.
Taiko https://taiko.gauge.org Taiko is a free and open source browser automation tool we'll be using in conjunction with Gauge.
VS Code https://code.visualstudio.com (OPTIONAL) You can use the text editor or IDE of your choice for these exercises. If you don't have one already installed, VS Code is a free and open source IDE.
Git https://git-scm.com (OPTIONAL) There are several projects that you'll be asked to git clone. However, you can also simply download the ZIP file without cloning the repos.
Caddy https://caddyserver.com (OPTIONAL) These exercises will run from any standard web server. If you don't have a web server already installed, Caddy is a nice, reliable HTTP/2 server that is easy to install.

Steps

Install the ConferenceWorks-Starter Project

  1. Download the ConferenceWorks starter project.
    This is a barebones website that you will build up over the course of several exercises.
  2. Start the webserver in the conferenceworks-starter directory.
    If you have Caddy installed, you can type caddy.
  3. Visit http://localhost:8888.
    You should see the welcome screen for the ConferenceWorks-Starter project.
