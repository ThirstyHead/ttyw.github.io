Tables and LocalStorage

In this section, we'll use an HTML <table> to lay out the main conference calendar of talks. Since the Schedule data is more volatile than the Speaker data, we'll call a Schedule microservice that returns JSON data and build the table dynamically.

To help build out the table data in a consistent manner, we'll use an HTML <template>.

We'll also look at browser-native storage options like SessionStorage and LocalStorage to cache the data locally once we've fetched it.
